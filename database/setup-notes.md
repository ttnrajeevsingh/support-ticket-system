# Database Setup Notes

## Database Choice
PostgreSQL via **Supabase** (hosted, free tier).

## Why Supabase
- No local PostgreSQL installation required
- Free tier sufficient for this project (500MB storage)
- Standard PostgreSQL — all SQL features work (GIN indexes, enums, triggers)
- Connection pooling built-in
- Prisma connects identically to any PostgreSQL host

## Schema Location
`server/prisma/schema.prisma` — this is the source of truth.

## Migration Files
`server/prisma/migrations/` — auto-generated by `prisma migrate dev`.

Two migrations:
1. `20260710115631_init` — creates tables, enums, indexes, relations
2. `20260710115710_add_fts_index` — adds GIN full-text search index

## Seed Data
`server/prisma/seed.ts` — run with `npx prisma db seed`.

Creates:
- 3 users (Alice Admin, Bob Agent, Carol Agent)
- 5 tickets (one per status)
- 3 comments

## Connection Setup

Both URLs use the **session-mode pooler** (port 5432 on the pooler host):
```
DATABASE_URL="postgresql://postgres.[ref]:[PASSWORD]@aws-0-[region].pooler.supabase.com:5432/postgres"
DIRECT_URL="postgresql://postgres.[ref]:[PASSWORD]@aws-0-[region].pooler.supabase.com:5432/postgres"
```

Port 6543 (transaction pooler) was blocked on my network, so both use port 5432 (session mode). This works for both Prisma Client (runtime) and Prisma Migrate (DDL).

## Commands

```bash
# Apply migrations to a fresh database
npx prisma migrate deploy

# Seed the database
npx prisma db seed

# Open Prisma Studio (visual DB browser)
npx prisma studio

# Validate schema syntax
npx prisma validate

# Generate Prisma Client after schema changes
npx prisma generate
```
